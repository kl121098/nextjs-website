d:T1628,<p><strong>Introduction</strong></p>
<p>Pi-Health is a comprehensive health and wellness application
designed to empower users in managing their physical and mental
well-being. The app aims to provide personalized health insights,
promote healthy habits, and facilitate communication between users
and healthcare professionals. Whether you're tracking your fitness
goals, monitoring vital signs, or seeking medical advice, Pi-Health
has you covered.</p>
<p><strong>Technology Stack</strong></p>
<p>Pi-Health leverages a robust technology stack to deliver a seamless
user experience. Here are the key components:</p>
<p>1.Front-End:
○ React: A popular JavaScript library for building user
interfaces.</p>
<p>○ Typescript: Provides static typing and enhances code
quality.</p>
<p>○ Chakra UI: A component library for creating accessible
and customizable UI components.</p>
<p>○ Axios: Handles HTTP requests to external APIs.</p>
<p>○ Tanstack React Query: Manages data fetching and
caching.</p>
<p>○ Tanstack React Router: Handles client-side routing.</p>
<p>○ Dexie: A lightweight IndexedDB wrapper for local data
storage.</p>
<p>○ Dexie React Hooks: Integrates Dexie with React
components.</p>
<p>○ Felte: A form library for handling form state and
validation.</p>
<p>○ Jotai: A state management library.</p>
<p>○ Zod: A TypeScript-first schema-driven validation library.</p>
<p>2.Back-End:
○ Rust: A systems programming language known for its
safety, performance, and expressive syntax.</p>
<p>○ Actix Web Framework: A high-performance, actor-
based web framework in Rust, employed for handling
HTTP requests and responses efficiently.</p>
<p>○ SQLite DB: SQLite is utilized as the back-end database
for server-side data storage.</p>
<p>○ JSON-RPC API: Implementation of JSON-RPC endpoints
facilitates structured communication between the front-
end and back-end.</p>
<p>○ Event Sourcing and CQRS: Integration of Event
Sourcing and CQRS concepts to improve the application's
scalability, auditability, and state management.</p>
<p><strong>Engine Separation Strategy</strong></p>
<p>The engine separation strategy in Pi-Health focuses on decoupling
the core application logic from external dependencies. Here's how
it's achieved:</p>
<p>● Components: The application is organized into reusable
components, promoting a component-based architecture.</p>
<p>● Features: Each feature (e.g., doctor details, patient vitals,
medication, appointments) resides in its own folder,
encapsulating related functionality.</p>
<p>● Routes: Separate route files define routes for each feature,
ensuring modularity and easy navigation.</p>
<p>● Types: Necessary types are defined in a dedicated folder,
promoting type safety and consistency.</p>
<p>● Services: Services handle API calls using Axios for
communication with the back-end.</p>
<p>● Query-Mutation-Services: Tanstack React Query is used for
mutation purposes (adding, updating, and deleting data) to the
server. The benefits of useMutation from Tanstack simplify
mutation handling.</p>
<p>● Database:
Client – Side : Dexie is used for storing data locally in a
database, avoiding network requests and resulting in faster
responses. Dexie React Hooks integrates the Dexie database
with React components, ensuring real-time data reflection.
Server – Side : SQLite was chosen as the back-end database
for server-side data storage. Its lightweight nature, ease of
integration, and ACID compliance make it suitable for
applications with moderate data storage and retrieval
requirements.</p>
<p><strong>Challenges and Solutions</strong></p>
<p>Client-Side
Synchronization:
Challenge: Syncing local data with the server.
Solution: Dexie's built-in synchronization mechanisms or
real-time techniques.
State Management:
Challenge: Handling complex state across features.
Solution: Jotai for global and local state management</p>
<p>Server-Side
Considerations:
While Rust and the selected technologies offer numerous
advantages, the adoption of a relatively new language
and framework may present challenges in terms of
available resources and expertise. However, these
challenges can be mitigated through training,
documentation, and community support.</p>
<p><strong>Achievements</strong></p>
<p>● Client-Side
Reliability: Separation of concerns promotes reliability.
High Performance: Dexie minimizes network requests.
User-Friendly: Well-organized features enhance the user
experience.
Offline Capability: Dexie enables offline usage.</p>
<p>● Server-Side
The integration of Rust, Actix Web, SQLite, JSON-RPC API, and
Event Sourcing/CQRS has resulted in a robust, scalable, and
performant back-end. The chosen technologies contribute to
the overall safety, efficiency, and maintainability of the
system, aligning with the project's objectives and ensuring a
positive user experience.
Key Learning</p>
<p>● Modularity Matters: Breaking down the application into
smaller, manageable pieces simplifies development.</p>
<p>● Client-Side Storage: Dexie improves responsiveness.</p>
<p>● Rust and Actix Web Framework for Safety and
Performance: Leveraging Rust's memory safety and Actix
Web's actor-based architecture ensures a secure and efficient
web application.</p>
<p>● Event Sourcing, CQRS, and Structured Communication
Drive Scalability: Integrating Event Sourcing and CQRS
principles, along with a JSON-RPC API, enhances scalability and
auditability.</p>
